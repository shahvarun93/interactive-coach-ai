Rate Limiting Cookbook

Common Algorithms

  • Fixed window – simple, but edge bunched at boundaries.
  • Sliding window – smoother, more precise.
  • Token bucket / leaky bucket – burst-friendly, steady average rate.

Where to Enforce Limits

  • API gateway / edge proxy.
  • Per-user, per-IP, per-API-key, or per-tenant.
  • Client-side guards for UX (optional, not security).

State & Storage

  • In-memory for single node; Redis or similar for distributed limits.
  • Key design: rate:{scope}:{id} (e.g., rate:user:123).

Interview Checklist

  • Clarify what is limited (requests/sec? per minute? per feature?).
  • Explain algorithm and why it fits the use case.
  • Discuss distributed coordination and race conditions.
  • Consider abuse patterns and how to monitor.
