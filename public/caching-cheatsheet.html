Caching Strategy Cheat Sheet

Core Concepts

  • Cache hit / miss / eviction
  • Read-through / Write-through / Write-back / Write-around
  • TTL, stale-while-revalidate
  • Cold start vs warm cache

When to Use Caching

  • Expensive reads (CPU, I/O, network)
  • Read-heavy workloads
  • Data that can tolerate slight staleness

Key Design Questions

  • What is the source of truth?
  • What is the cache key and value?
  • What is the invalidation strategy?
  • What is the eviction policy (LRU, LFU, etc.)?
  • What happens when the cache fails?

Typical Patterns

  • Local (in-process) cache for ultra-low latency.
  • Distributed cache (e.g., Redis) for shared state.
  • CDN / edge cache for static assets and public APIs.

Interview Checklist

  • State what you cache and why.
  • Explain invalidation clearly.
  • Cover failure modes and fallback to DB.
  • Discuss hot keys and cache stampede mitigation.
