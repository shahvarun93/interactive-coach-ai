Search Ranking Primer

Basic Concepts

  • Index vs document store.
  • BM25 / TF-IDF style term relevance (high level).
  • Exact match, prefix match, fuzzy search.

Signals for Ranking

  • Text relevance (query vs document fields).
  • Freshness / recency.
  • Popularity (clicks, purchases, views).
  • Personalization (user history, location).

Architecture Sketch

  • Write path: ingest → normalize → index.
  • Read path: query parsing → candidate retrieval → ranking → post filters.

Interview Checklist

  • Clarify what “relevance” means for this product.
  • Explain how you’d index and which fields matter most.
  • Mention ranking factors and tuning (offline vs online metrics).
  • Consider latency and cache for popular queries.
